## Обзор экранов

Нижняя панель содержит пять вкладок:

| Иконка | Экран | Назначение |
|--------|-------|-----------|
| Дом | **Главная** | Общий баланс, карусель счетов, сводка за месяц, последние операции |
| Карточка | **Счета** | Список всех счетов, создание новых, перевод между счетами |
| Копилка | **Бюджеты** | Лимиты расходов по категориям и счетам |
| Столбики | **Аналитика** | История операций с фильтрами, диаграммы, разбивка по категориям и счетам |
| График | **Прогноз** | Прогноз балансов с учётом регулярных платежей и вкладов |

Шестерёнка в правом верхнем углу главного экрана открывает **Настройки** (категории, безопасность, экспорт, резервные копии).

Плавающая кнопка **«+»** над нижней панелью открывает форму добавления операции.

## Счета

### Типы счетов

| Тип | Когда использовать |
|-----|--------------------|
| **Карта** | Банковская карта, расчётный счёт |
| **Наличные** | Кошелёк, наличные деньги |
| **Накопительный счёт** | Вклад без фиксированной даты окончания (проценты начисляются бессрочно) |
| **Срочный вклад (Депозит)** | Вклад с датой окончания; прогноз автоматически учитывает выплату |
| **Инвестиции** | Брокерский счёт или другие инвестиции |
| **Другое** | Всё остальное |

### Как добавить счёт

1. Перейди на вкладку **«Счета»** (иконка карточки).
2. Нажми **«+»** (FAB).
3. Заполни:
   - **Название** — любое, например «Тинькофф» или «Кошелёк»
   - **Тип** — выбери из списка выше
   - **Валюта** — RUB, USD, EUR, GBP, CNY, KZT; рядом с кодом отображается символ (₽, $, €, £, ¥, ₸)
   - **Цвет** — выбери из палитры; цвет отображается в карусели на главной и в аналитике
   - **Иконка** — выбери подходящую иконку (банк, кошелёк, карта и т.д.)
   - **Начальный баланс** — текущий остаток, с которого начинается учёт
4. Нажми **✓** (сохранить).

### Срочный вклад — дополнительные параметры

При типе **«Депозит»** появляется секция вклада:

| Поле | Описание |
|------|----------|
| **Сумма вклада** | Внесённая сумма (principal) |
| **Процентная ставка** | Годовая ставка в % |
| **Дата открытия** | Когда открыт вклад |
| **Дата окончания** | Когда вклад закрывается |
| **Капитализация** | Вкл/выкл; при включении проценты добавляются к телу |
| **Период капитализации** | Ежемесячно / ежеквартально / ежегодно |
| **Счёт для выплаты** | Куда перечислить сумму при окончании (необязательно) |

Поддерживается **ступенчатая ставка**: можно добавить несколько периодов с разными процентами (например, первый месяц 15%, далее 14%). Чтобы переключиться обратно на одну ставку — кнопка «Использовать единую ставку».

В форме сразу виден **живой прогноз** итоговой суммы.

### История начислений вклада

На детальном экране вклада (нажми на счёт-депозит) отображается **лента событий начислений**: каждая строка показывает дату, сумму начисленных процентов и нарастающий итог. Переключатель **«Показать по дням / по месяцам»** меняет сегментацию. Пролистай ленту, чтобы увидеть всю историю роста вклада.

### Накопительный счёт

Тип **«Накопительный счёт»** — это вклад без даты окончания. Баланс показывается как
«principal + накопленные проценты на сегодня». Прогноз учитывает начисление процентов
вплоть до выбранной даты.

### Редактирование и архивирование

- **Нажми на счёт** → откроется детальный экран.
- **Меню ⋮ на карточке счёта** → «Редактировать» или «Архивировать».
- **Свайп влево** по счёту в списке — удаление (необратимо, удалятся все транзакции этого счёта).
- Нельзя удалить счёт, на который есть незавершённые прогнозы — сначала закрой вклад.

### Изменение порядка

Иконка ⇅ в правом верхнем углу экрана «Счета» открывает режим «Порядок счетов». В нём можно перемещать любой счёт стрелками ↑/↓. Нажми ✓ для сохранения или ← для отмены. Порядок применяется к карусели счетов на главном экране и внутри списков на других экранах.

### Архив и восстановление

Архивирование скрывает счёт из основных списков, карусели и сводок, но **не удаляет** его — транзакции этого счёта остаются в базе, просто не видны.

- Чтобы увидеть архивные счета, нажми иконку «глаз» в правом верхнем углу экрана «Счета».
- Архивный счёт отмечен меткой «архив» рядом с названием.
- В меню ⋮ архивного счёта появляется пункт **«Восстановить»** — счёт снова становится активным, его транзакции возвращаются в историю и аналитику.

### Перевод между счетами

1. Открой любой счёт.
2. Нажми **«Перевод»**.
3. Выбери счёт-получатель и сумму.

Перевод фиксируется как транзакция типа **TRANSFER** и корректирует балансы обоих счетов.

> **Ограничение:** перевод между счетами в разных валютах невозможен — оба счёта должны быть в одной валюте.

## Транзакции

### Добавление операции

Нажми **«+»** в нижней панели. Форма содержит:

1. **Тип операции** — Доход / Расход / Перевод / В накопления
2. **Сумма** — обычное текстовое поле, системная цифровая клавиатура. Можно поставить курсор в середину числа, копировать/вставлять. Максимум 2 знака после запятой.
3. **Счёт** — откуда списывается (или куда начисляется) сумма
4. **Категория** — выбери из списка или создай новую
5. **Дата** — по умолчанию сегодня; нажми чтобы изменить
6. **Время** — необязательно; формат ЧЧ:ММ. Если указано, отображается под суммой в списке последних операций и в истории. Операции с одинаковой датой сортируются по времени.
7. **Заметка** — необязательный комментарий
8. **Повтор** — настрой регулярное правило (см. ниже)

Нажми **✓** в правом верхнем углу для сохранения.

### Редактирование и удаление

- В **истории** нажми на транзакцию → откроется форма редактирования.
- В **истории** свайп влево → удалить одну операцию.
- Включи **режим выбора** (зажми транзакцию) → выбери несколько → удали пакетом. Пакетное удаление требует подтверждения.

**Удаление повторяющейся транзакции** — если операция входит в серию, появится диалог выбора области:
| Вариант | Что происходит |
|---------|---------------|
| **Только эту** | Удаляется одна операция, серия продолжается |
| **Эту и все будущие** | Серия останавливается на этой дате, прошлые операции сохраняются |
| **Остановить серию** | Правило удаляется, все прошлые операции сохраняются |

### Типы операций

| Тип | Что делает |
|-----|-----------|
| **Доход** | Увеличивает баланс счёта |
| **Расход** | Уменьшает баланс счёта |
| **Перевод** | Уменьшает один счёт, увеличивает другой |
| **В накопления** | Списывает с обычного счёта и записывает как «экономию» (тип расхода без категории трат) |

## Категории

Категории используются для классификации доходов и расходов.

### Как добавить / изменить категорию

**Настройки → Категории** или нажми **«+»** в CategoryPicker при создании транзакции.

Форма:
1. **Название** — произвольное
2. **Тип** — Доход / Расход / Перевод
3. **Цвет** — 19 вариантов
4. **Иконка** — 32 варианта (еда, транспорт, здоровье, дом, бар, одежда, топливо, дети, музыка, кино, красота, аптека, интернет, природа, благотворительность, ремонт, подписки, мото и др.)
5. **Родительская категория** — необязательно; создаёт подкатегорию (до трёх уровней: раздел → категория → подкатегория)

## Бюджеты

Бюджет — это лимит расходов на выбранный период по выбранным категориям и счетам.

### Как создать бюджет

**Настройки → Бюджеты → «+»**

1. **Лимит** — максимальная сумма расходов
2. **Период** — Ежемесячно или Еженедельно
3. **Категории** — выбери «Все категории» или отметь конкретные (можно несколько)
4. **Счета** — выбери «Все счета» или отметь конкретные (можно несколько)

### Как читать карточку бюджета

```
Еда, Транспорт          [редактировать] [удалить]
Все счета
1 234,00 / 5 000,00
████████░░░░░░░░░░░░  (прогресс-бар зелёный или красный)
```

- **Три цвета** прогресс-бара: зелёный → жёлтый → красный. Пороги настраиваются.
- Сумма слева — потрачено за текущий период, справа — лимит; рядом — валюта бюджета
- Ниже — доля в процентах (цвет совпадает с баром)
- **Бейдж на вкладке «Бюджеты»** в нижней панели: жёлтый — если какой-то бюджет перешёл жёлтый порог; красный — если перешёл красный

### Пороги цветов

Глобальные дефолты: **Настройки → Бюджеты → Пороги по умолчанию** (по умолчанию 70% жёлтый, 90% красный). В диалоге создания/редактирования бюджета есть опциональные поля «Жёлтый порог» и «Красный порог» — если заполнены, используются вместо глобальных.

Бюджет привязан к **одной валюте**: расходы по счетам в других валютах не учитываются. Для USD и RUB создавай отдельные бюджеты.

Удаление бюджета (иконка 🗑) требует подтверждения: «Удалить бюджет? — Удалить / Отмена».

## Аналитика и история

### История операций

Вкладка **«История»** (иконка списка).

**Поиск** (иконка лупы в правом верхнем углу) — открывает поле ввода прямо в шапке экрана. Ищет по заметке, категории, названию счёта и счёту-получателю (для переводов). Кнопка ← закрывает поиск и сбрасывает запрос.

**Фильтры** (кнопка воронки):
- Период: свой диапазон дат
- Сумма: от / до (оба поля необязательны)
- Тип: Доход / Расход / Перевод / В накопления (мульти-выбор)
- Счёт: один или несколько (с валютой и группировкой по типу)
- Категория: одна или несколько

**Режим выбора** — зажми транзакцию, отмети нужные, удали пакетом.

**Группировка** — операции сгруппированы по дням, в заголовке дня — итоговая сумма.

### Аналитика

Вкладка **«Аналитика»**.

**Переключатель периода** — стрелки влево/вправо для навигации по месяцам.

**Переключатель режима** — «По категориям» или «По счетам».

#### Режим «По категориям»

- **Круговая диаграмма** расходов
- **Топ расходов** — список категорий с прогресс-баром, долей и количеством операций
- **Топ доходов** — аналогично
- **Среднедневной расход** — в нижней части

<p align="center">
  <img src="screenshots/analytics_categories.jpg" width="280" alt="Аналитика — режим «По категориям»" />
</p>

Нажми на категорию → откроется **детализация**: тренд расходов по категории (столбчатый график) и список подкатегорий с суммами. Родительские категории показывают **суммарный rollup** — включая все вложенные подкатегории.

#### Режим «По счетам»

Список счетов с суммой расходов/доходов, прогресс-баром (в цвете счёта) и долей в процентах. Нажми на счёт → раскроется **детализация по категориям** для этого счёта.

#### Столбчатый график трендов

В верхней части аналитики — **столбчатый график**: зелёные столбики — доходы, красные — расходы. Над каждым столбиком отображается количество операций. Кнопки **4 нед / 8 нед / 6 мес / 12 мес** меняют диапазон отображения.

## Прогнозы

Вкладка **«Прогнозы»** (иконка графика).

### Как пользоваться

1. Выбери **дату прогноза** (быстрые пресеты: +1 мес, +3 мес, +6 мес, +1 год, или точная дата).
2. Приложение рассчитывает, каким будет баланс каждого счёта к этой дате.

### Что учитывается

| Событие | Откуда берётся |
|---------|----------------|
| Регулярные платежи/доходы | Правила повторяющихся операций |
| Окончание срочного вклада | Дата окончания вклада + итоговая сумма |
| Проценты по накопительному счёту | Начисление до выбранной даты |
| Капитализация по вкладу | Рост баланса по формуле сложного процента (если включена) |

Для **срочного вклада без капитализации** баланс остаётся на уровне суммы вклада до даты окончания — проценты приходят одномоментно в день закрытия и попадают на счёт-получатель (или на сам вклад, если счёт-получатель не указан). Прогноз это отражает: до `endDate` график плоский, затем скачок.

### График Cash Flow

Столбчатый график доходов и расходов по неделям на горизонте прогноза. Позволяет увидеть «проблемные» недели с дефицитом до того, как они наступят.

### Сценарии

Кнопка **«Сценарии»** открывает панель, где можно добавить разовые или регулярные запланированные события — доходы и расходы — без сохранения в основную базу. Они учитываются только в расчётах прогноза на текущей сессии. Это позволяет проверить «что будет, если…» без изменения данных.

### Запланированные повторяющиеся события

В панели сценариев можно создавать **повторяющиеся запланированные события** (частота, интервал, дата окончания). При удалении события с правилом появляется диалог выбора области: только это / это и будущие / вся серия — аналогично обычным транзакциям.

### Итоговая таблица

Каждый счёт — строка с иконкой, цветом счёта и прогнозным балансом. Ниже — итоги по валютам.

### Лента событий

Хронологический список всех событий между сегодня и выбранной датой: когда, что, на какую сумму и какой счёт затронут. Иконка и цвет счёта помогают быстро ориентироваться.

## Повторяющиеся операции

### Создание

При создании транзакции нажми **«Повтор»**:

| Поле | Описание |
|------|---------|
| **Частота** | Ежедневно / Еженедельно / Ежемесячно / Ежегодно |
| **Интервал** | Каждые N единиц (например, каждые 2 недели) |
| **Дата окончания** | Когда прекратить генерацию (необязательно) |

Приложение автоматически создаёт транзакции в нужные даты (в том числе за пропущенные, если приложение было закрыто).

### Управление правилами

**Настройки → Повторяющиеся операции** — список всех активных серий.

Каждая карточка показывает: иконку и имя категории, сумму, частоту, следующую дату генерации.

Нажми на карточку для просмотра деталей (когда создано, сколько транзакций уже сгенерировано, привязанный счёт). Из деталей можно:
- **Остановить серию** — правило удаляется, все уже созданные операции остаются.
- **Изменить** — перейти в форму редактирования первой транзакции серии.

## Настройки

### Основные

| Настройка | Описание |
|-----------|---------|
| **Валюта по умолчанию** | Валюта, которая показывается первой в мульти-валютном балансе |
| **Тема оформления** | Системная / Светлая / Тёмная |

### Категории

**Настройки → Категории** — создание, редактирование, удаление категорий для операций.

### Повторяющиеся операции

**Настройки → Повторяющиеся операции** — просмотр и остановка активных серий (см. раздел [Повторяющиеся операции](#повторяющиеся-операции)).

### Бюджеты

**Настройки → Бюджеты** — лимиты расходов по категориям и счетам.

### Экспорт в CSV

**Настройки → Резервная копия → Экспорт в CSV**

Выгрузка всех операций в файл `.csv` с разделителем `;` (совместим с Excel и Google Таблицами). Первая строка — заголовки, кодировка UTF-8 с BOM.

### Журнал ошибок

**Настройки → О приложении → Отправить логи**

Если приложение падало — здесь можно поделиться файлом с подробностями ошибки для диагностики. Отправляется через стандартный системный шаринг (Telegram, email и т.д.).

## Безопасность и резервные копии

### PIN-код и биометрия

Приложение защищено **4-значным PIN-кодом**, который устанавливается при первом запуске. PIN верифицируется через Argon2id, а ключ базы данных хранится в Android Keystore (аппаратно, без возможности извлечения). **Без PIN данные недоступны**.

При 5 неверных вводах подряд включается экспоненциальная блокировка (30 с, 60 с, …, максимум 24 ч). Данные при этом не удаляются.

### Смена PIN-кода

**Настройки → Безопасность → Сменить PIN**

Введи текущий PIN и дважды — новый. После смены биометрия автоматически отключается и требует повторной настройки.

### Пароль резервных копий

Пароль для резервных копий **отдельный** от PIN — он задаётся при первом создании копии и используется исключительно для шифрования файла `.mkbak`. Один и тот же файл можно восстановить на другом устройстве, зная только этот пароль.

**Менеджеры паролей** — поля пароля резервных копий объявлены как autofill-совместимые. Google Password Manager (встроен в Android) или любой сторонний (Bitwarden, 1Password) предложит сохранить пароль при первом создании.

### Биометрия

**Настройки → Безопасность → Биометрическая разблокировка**

Вход по отпечатку пальца или Face ID вместо ввода PIN. Биометрия — только для разблокировки приложения; резервные копии всегда шифруются отдельным паролем.

Биометрия инвалидируется автоматически, если:
- Сменишь PIN в приложении
- Добавишь новый отпечаток в настройках телефона

В обоих случаях нужно включить биометрию заново.

### Резервные копии

**Настройки → Резервная копия**

#### Создать резервную копию

Сохраняет зашифрованный файл `.mkbak`. Без пароля резервной копии файл открыть невозможно.

<p align="center">
  <img src="screenshots/backup.jpg" width="280" alt="Экран «Резервная копия»" />
</p>

Рекомендуется хранить копию:
- В Telegram «Избранное»
- Google Drive / Яндекс.Диск
- На компьютере

#### Восстановить из резервной копии

Выбери `.mkbak`-файл и введи пароль резервной копии, который был **в момент её создания** (не обязательно совпадает с текущим).

#### Перенос на новое устройство

1. Создай резервную копию на старом устройстве.
2. Установи приложение на новом.
3. Пройди первый запуск (создай любой пароль).
4. **Настройки → Резервная копия → Восстановить** → выбери файл → введи **пароль от этой копии**.
5. Приложение перезапустится с восстановленными данными.

## Быстрые ответы

**Можно ли вести несколько валют одновременно?**  
Да. Каждый счёт имеет свою валюту. На главной экране баланс показывается отдельной строкой для каждой валюты.

**Почему нет синхронизации с другими устройствами?**  
Приложение хранит данные только локально — это гарантирует, что финансовые данные не попадут на сторонние серверы. Для переноса используй резервные копии.

**Что будет, если удалить приложение?**  
Данные будут удалены вместе с приложением. Восстановить их можно только из предварительно созданной резервной копии.

**Что будет, если забыть PIN?**  
PIN верифицируется на устройстве и нигде не хранится в открытом виде. Если PIN забыт — восстановить доступ к данным невозможно без резервной копии. Единственная защита — регулярные резервные копии с отдельным паролем, который ты помнишь.

**Что будет, если забыть пароль от резервной копии?**  
Файл `.mkbak` зашифрован и без пароля не восстановить. Храни пароль в менеджере паролей.

**Что такое «В накопления» тип операции?**  
Это расход, который учитывается отдельно от обычных трат. Полезно, чтобы не смешивать «откладываю деньги» и «трачу деньги» в аналитике.
